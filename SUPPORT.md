# Support

We want Velatis Charts to be straightforward to run and build upon.

## Community support

Use GitHub Discussions for setup questions, extension design, integrations, and
ideas. Use GitHub Issues for reproducible defects and focused feature requests.
Maintainers will help when time permits, and useful documentation fixes are
always welcome.

Please include:

- the exact package version or commit;
- Node.js, pnpm, operating system, and CPU architecture;
- the command you ran and complete non-secret error output;
- a minimal chart state with personal data replaced by synthetic values.

Never post a real person's birth data without their permission.

## Security and private reports

Do not open public issues for vulnerabilities. Follow
[SECURITY.md](./SECURITY.md).

## Boundaries

Community support is best effort. We cannot operate third-party deployments,
guarantee response times, provide legal advice, or debug undisclosed proprietary
modifications. Commercial support can be discussed separately in the future,
but the open project will remain usable without it.
